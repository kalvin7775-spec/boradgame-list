# 我的桌遊櫃 — 穩定版（Cloudflare Pages）

這個版本附帶一支**後端代理 function**（`functions/api/bgg.js`），由伺服器端代為向 BGG 取資料，網頁前端只跟自己的網站同源溝通，因此**完全沒有 CORS 問題、不依賴任何公開代理**，穩定可靠。

## 資料夾結構（請保持不變）

```
桌遊櫃-cloudflare/
├── index.html              ← 網頁本體
└── functions/
    └── api/
        └── bgg.js          ← BGG 代理 function（網址 /api/bgg）
```

`functions/` 這個資料夾名稱與層級是 Cloudflare 的固定規則，**不要更改**，否則代理不會生效。

---

## 部署方式 A：直接上傳資料夾（最快，免 Git）

1. 註冊 / 登入 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左側選 **Workers & Pages** → **Create** → 切到 **Pages** 分頁 → **Upload assets**
3. 專案命名（例如 `my-boardgame-shelf`）
4. 把整個 `桌遊櫃-cloudflare` 資料夾裡的內容拖進去上傳（要包含 `functions` 資料夾）
5. 按 **Deploy site**
6. 完成後會給你一個網址：`https://你的專案名.pages.dev`

> 注意：拖拉上傳時，要上傳的是「資料夾內的內容」（index.html 與 functions/），而不是把外層資料夾整包丟進去造成多一層路徑。

---

## 部署方式 B：連結 GitHub（之後更新最方便）

1. 把 `桌遊櫃-cloudflare` 資料夾的內容推到一個 GitHub repository
2. Cloudflare → **Workers & Pages** → **Create** → **Pages** → **Connect to Git**
3. 選擇該 repository
4. 建置設定全部留空（這是純靜態站 + Functions，不需要建置指令）
   - Build command：留空
   - Build output directory：留空或填 `/`
5. **Save and Deploy**

日後只要 push 到 GitHub，Cloudflare 會自動重新部署。

---

## 使用方式

1. 打開你的 `.pages.dev` 網址
2. 上方輸入桌遊**英文原名**（如 `Catan`、`Nemesis`），按 **Enter**
3. 從下拉清單選你擁有的版本 → 自動加入櫃子
4. 點任一盒子查看：封面、設計師、出版社、遊玩人數、遊戲時間、建議年齡、難度（複雜度）、BGG 評分與排名
5. 右上角可在自己的收藏中快速查找
6. 收藏資料存在你瀏覽器的 localStorage，下次開啟還在

## 常見問題

| 問題 | 解法 |
|------|------|
| 搜尋一直失敗 | 確認 `functions/api/bgg.js` 有一起上傳；在網址後加 `/api/bgg?endpoint=search&qs=query%3Dcatan` 測試是否回傳 XML |
| 圖片沒出現 | BGG 圖片網址偶爾較慢，重新整理即可 |
| 想換不同電腦看收藏 | localStorage 綁瀏覽器，換裝置不會同步（如需跨裝置同步要另做帳號/雲端，可再找我加） |
| 著作權 | 封面為連結至 BGG 圖片、資料為事實性資訊，供個人收藏管理用途 |
